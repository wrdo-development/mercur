## Introduction

[Medusa Cloud](https://cloud.medusajs.com) is a dedicated hosting service for Medusa-based applications. It automatically detects changes on your repository, pulls and builds the code, and provisions a database, Redis, and file storage.

Head to the [Medusa Cloud signup page](https://cloud.medusajs.com/signup) to get started. After registration is complete and your account is active, authorize the Medusa Cloud GitHub app in your organization or personal account.

<Info>
  Medusa Cloud builds the project with the Dockerfile in `packages/api`, which
  bundles the backend, the admin panel, and the vendor panel into a single
  container. Only the storefront needs to be deployed separately.
</Info>

## Prerequisites

- A Mercur project created with [`@mercurjs/cli`](/getting-started/installation) and pushed to a GitHub repository
- A [Medusa Cloud](https://cloud.medusajs.com) account with the GitHub app authorized
- API keys for any third-party services you plan to use (Stripe, Resend, Algolia, etc.)

## Setup

1. On the main dashboard view, click the **Create project** button.
2. Thanks to the GitHub integration, Medusa Cloud can access your codebase and automatically pull the code. Choose your Mercur repository from the list and click **Continue**.
   ![Choose repository](/images/cloud_2.png)
3. Under project details, choose a name for your project. You can also pick a custom subdomain.
   ![Basic project settings](/images/cloud_3.png)
4. Head to the **Build details** section. Under **Project root directory** enter `/packages/api`. Then, choose the first admin user credentials.
   ![Build details](/images/cloud_4.png)
5. The last configuration step is environment variables. Medusa Cloud automatically provisions a database, Redis, and file storage, so you don't have to configure them.
   ![Environment variables](/images/cloud_5.png)

Only the following variables should be set:

```bash
# CORS (point to the URLs that will host your panels and storefront)
STORE_CORS=
ADMIN_CORS=
VENDOR_CORS=
AUTH_CORS=

# Secrets
JWT_SECRET=
COOKIE_SECRET=

# Vendor panel URL used in seller emails and onboarding flows
MERCUR_VENDOR_URL=

# Stripe (customer payments + Mercur payout provider)
STRIPE_API_KEY=
STRIPE_WEBHOOK_SECRET=
STRIPE_PAYOUT_WEBHOOK_SECRET=

# Email (optional, e.g. Resend)
RESEND_API_KEY=
RESEND_FROM_EMAIL=

# Search (optional)
ALGOLIA_APP_ID=
ALGOLIA_API_KEY=
```

<Info>
  For Stripe setup, including which keys go where and how to wire both webhooks,
  see the [Stripe Connect integration
  guide](/how-to-guides/stripe-connect-integration).
</Info>

6. Click **Create**. The build and deploy process will start and can take a few minutes to complete.

You can change these variables later in environment settings, and switch the deployment branch at any time. Medusa Cloud will automatically pull and rebuild your environment whenever a new commit is pushed to the selected branch.

![ENV settings](/images/cloud_6.png)

## After deployment

Once the build succeeds, a single container exposes the API and both dashboards:

- The API is available at `https://<your-subdomain>.medusajs.app`.
- The Admin Panel is available at `https://<your-subdomain>.medusajs.app/dashboard`.
- The Vendor Panel is available at `https://<your-subdomain>.medusajs.app/seller`.
- Point your storefront deployment at the API URL via its `VITE_MEDUSA_BACKEND_URL` (or equivalent) build-time variable.

The dashboard routes are configured in `packages/api/medusa-config.ts` under the `admin-ui` and `vendor-ui` modules — change `path` there to remap them.
